## Core Workflow

The plugin centers around a simple workflow cycle using the `ce:*` commands:

<Steps>
  <Step title="Plan">
    Use `/ce:plan` to transform feature descriptions into well-structured plans
  </Step>
  <Step title="Work">
    Use `/ce:work` to execute plans systematically while maintaining quality
  </Step>
  <Step title="Review">
    Use `/ce:review` to perform exhaustive multi-agent code reviews
  </Step>
  <Step title="Compound">
    Use `/ce:compound` to document solved problems for future reference
  </Step>
</Steps>

See the [Workflow Guide](/plugin/workflow) for detailed documentation.

## Installation

The compound-engineering plugin is **bundled with Claude Code** and enabled by default. No installation required.

<Tabs>
  <Tab title="Claude Code">
    The plugin is bundled and enabled by default:

    ```bash
    # Already installed - just start using it
    /ce:plan
    ```

    To install manually:

    ```bash
    claude /plugin install compound-engineering
    ```
  </Tab>
  
  <Tab title="Cursor">
    Convert and install for Cursor:

    ```bash
    bunx @every-env/compound-plugin install compound-engineering --to cursor
    ```
  </Tab>
  
  <Tab title="OpenCode">
    Convert and install for OpenCode:

    ```bash
    bunx @every-env/compound-plugin install compound-engineering --to opencode
    ```

    Output location: `~/.config/opencode/`
  </Tab>
  
  <Tab title="Other IDEs">
    The plugin supports conversion to multiple formats:

    ```bash
    # Windsurf
    bunx @every-env/compound-plugin install --to windsurf

    # Factory Droid
    bunx @every-env/compound-plugin install --to droid

    # Gemini CLI
    bunx @every-env/compound-plugin install --to gemini

    # Install to all detected tools
    bunx @every-env/compound-plugin install --to all
    ```
  </Tab>
</Tabs>

## Philosophy

The plugin embodies **compound engineering** - a development philosophy where each unit of work makes future work easier, not harder.

Traditional development accumulates technical debt:
```
Work → Ship → Repeat → Debt accumulates → Slows down
```

Compound engineering creates compounding knowledge:
```
Work → Document → Reuse → Knowledge compounds → Speeds up
```

**Example**: The first time you solve an N+1 query issue takes 30 minutes of research. Document it with `/ce:compound`, and the next occurrence takes 2 minutes.

Read more in the [Philosophy Guide](/plugin/philosophy).

## Quick Start

Get started with the core workflow:

<Steps>
  <Step title="Explore an idea">
    ```bash
    /ce:brainstorm "Add user authentication"
    ```
    Collaborative dialogue to refine requirements and explore approaches.
  </Step>

  <Step title="Create a plan">
    ```bash
    /ce:plan "Add user authentication"
    ```
    Generates a structured plan with research, acceptance criteria, and implementation details.
  </Step>

  <Step title="Execute the work">
    ```bash
    /ce:work docs/plans/2026-03-03-feat-user-authentication-plan.md
    ```
    Systematically implements the plan with continuous testing and quality checks.
  </Step>

  <Step title="Review the code">
    ```bash
    /ce:review 123
    ```
    Multi-agent code review with security, performance, and architecture analysis.
  </Step>

  <Step title="Compound the knowledge">
    ```bash
    /ce:compound
    ```
    Documents the solution for future reference in `docs/solutions/`.
  </Step>
</Steps>

## Known Issues

### MCP Servers Not Auto-Loading

**Issue**: The bundled Context7 MCP server may not load automatically when the plugin is installed.

**Workaround**: Manually add it to your project's `.claude/settings.json`:

```json
{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    }
  }
}
```

Or add it globally in `~/.claude/settings.json` for all projects.
